                                <h1 id="integration-with-spatie-packages">Integration with Spatie packages</h1>

<h2 id="laravel-activitylog"><strong>laravel-activitylog</strong></h2>

<blockquote>
  <p>Note: The package requires logged models to have integer IDs. We recommend extra security measures when using integer IDs for tenants. Because the IDs become enumerable, they get vulnerable to enumeration attacks (which UUIDs are safe against).
  For example, to use the LogsActivity trait on the Tenant model, modify the model to have an integer ID.</p>
</blockquote>

<h3 id="for-the-tenant-app">For the tenant app:</h3>

<ul>
<li>Set the `database_connection` key in `config/activitylog.php` to `null`. This makes activitylog use the default connection.</li>
<li>Publish the migrations and move them to `database/migrations/tenant`. (And, of course, don't forget to run `artisan tenants:migrate`.)</li>
</ul>

<h3 id="for-the-central-app">For the central app:</h3>

<ul>
<li>Set the `database_connection` key in `config/activitylog.php` to the name of your central database connection.</li>
</ul>

<h2 id="laravel-permission"><strong>laravel-permission</strong></h2>

Install the package like usual, but publish the migrations and move them to `migrations/tenant`:



```
<code>php artisan vendor:publish --provider="Spatie\Permission\PermissionServiceProvider" --tag="migrations"
mv database/migrations/*_create_permission_tables.php database/migrations/tenant
</code>
```


Next, add the following listeners to the `TenancyBootstrapped` and `TenancyEnded` events to `events()` in your `TenancyServiceProvider`:



```
<code class="language-php">Events\TenancyBootstrapped::class =&gt; [
    function (Events\TenancyBootstrapped $event) {
        $permissionRegistrar = app(\Spatie\Permission\PermissionRegistrar::class);
        $permissionRegistrar-&gt;cacheKey = 'spatie.permission.cache.tenant.' . $event-&gt;tenancy-&gt;tenant-&gt;getTenantKey();
    }
],

Events\TenancyEnded::class =&gt; [
    function (Events\TenancyEnded $event) {
        $permissionRegistrar = app(\Spatie\Permission\PermissionRegistrar::class);
        $permissionRegistrar-&gt;cacheKey = 'spatie.permission.cache';
    }
],
</code>
```


Alternatively, create a bootstrapper:



```
<code class="language-php">class SpatiePermissionsBootstrapper implements TenancyBootstrapper
{
    public function __construct(
        protected PermissionRegistrar $registrar,
    ) {}

    public function bootstrap(Tenant $tenant): void
    {
        $this-&gt;registrar-&gt;cacheKey = 'spatie.permission.cache.tenant.' . $tenant-&gt;getTenantKey();
    }

    public function revert(): void
    {
        $this-&gt;registrar-&gt;cacheKey = 'spatie.permission.cache';
    }
}
</code>
```


The reason for this is that spatie/laravel-permission caches permissions &amp; roles to save DB queries, which means that we need to separate the permission cache by tenant. We also need to reset the cache key when tenancy ends so that the tenant's cache key isn't used in the central app.



<h2 id="laravel-medialibrary"><strong>laravel-medialibrary</strong></h2>

To generate the correct media URLs for tenants, create a custom URL generator class extending `Spatie\MediaLibrary\Support\UrlGenerator\DefaultUrlGenerator` and override the `getUrl()` method:



```
<code class="language-php">use Spatie\MediaLibrary\Support\UrlGenerator\DefaultUrlGenerator;

class TenantAwareUrlGenerator extends DefaultUrlGenerator
{
    public function getUrl(): string
    {
        $url = asset($this-&gt;getPathRelativeToRoot());

        $url = $this-&gt;versionUrl($url);

        return $url;
    }
}
</code>
```


Then, change the `url_generator` in the media-library config to the custom one (make sure to import it):



```
<code class="language-php">'url_generator' =&gt; TenantAwareUrlGenerator::class,
</code>
```


<h2 id="laravel-login-link"><strong>laravel-login-link</strong></h2>

After installing the package, publish the config file:



```
<code class="language-sh">php artisan vendor:publish --tag="login-link-config"
</code>
```


After that, add your <a href="https://tenancyforlaravel.com/docs/v3/tenant-identification">tenant identification</a> middlewares to the config file's `middleware` key:



```
<code class="language-php">'middleware' =&gt; [
    'web',
    // Add whatever tenant identification middlewares you use in your tenant routes
    InitializeTenancyByDomain::class,
    PreventAccessFromCentralDomains::class,
],
</code>
```
