                                <h1 id="console-commands">Console commands</h1>

The package comes with some useful artisan commands.



Tenant-aware commands run for all tenants by default. The commands also have the `--tenants` option which lets you specify IDs of the tenants for which the command will run.



<blockquote>
  Note: To include multiple tenants using CLI, you can use multiple `--tenants=&lt;...&gt;` options. If you're calling the command using `Artisan::call()`, `--tenants` has to be an array.


</blockquote>

<h2 id="migrate"><strong>Migrate</strong> (tenant-aware)</h2>

The `tenants:migrate` command migrates databases of your tenants.



```
<code>php artisan tenants:migrate --tenants=8075a580-1cb8-11e9-8822-49c5d8f8ff23
</code>
```


<blockquote>
  Note: By default, the migrations should be in database/migrations/tenant. If you wish to use a different path, you may use the `--path` option. You can also specify the default parameters for the command <a href="https://tenancyforlaravel.com/docs/v3/configuration#migration-parameters">in the tenancy config</a>


</blockquote>

<h2 id="rollback-and-seed"><strong>Rollback &amp; seed</strong> (tenant-aware)</h2>

<ul>
<li>Rollback: `tenants:rollback`</li>
<li>Seed: `tenants:seed`</li>
</ul>

<blockquote>
  Note: You can configure the default parameters for `tenants:seed` (e.g. use a custom tenant seeder) in <a href="https://tenancyforlaravel.com/docs/v3/configuration#seeder-parameters">the tenancy config</a>


</blockquote>

<h2 id="migrate-fresh"><strong>Migrate fresh</strong> (tenant-aware)</h2>

This package also offers a simplified, tenant-aware version of `migrate:fresh`. It runs `db:wipe` and `tenants:migrate` on the tenant's database.



You may use it like this:



```
<code>php artisan tenants:migrate-fresh --tenants=8075a580-1cb8-11e9-8822-49c5d8f8ff23
</code>
```


<h2 id="run"><strong>Run</strong> (tenant-aware)</h2>

You can use the `tenants:run` command to run your own commands for tenants.



If your command's signature were `email:send {--queue} {--subject=} {body}`, you would run this command like this:



```
<code>php artisan tenants:run email:send --tenants=8075a580-1cb8-11e9-8822-49c5d8f8ff23 --option="queue=1" --option="subject=New Feature" --argument="body=We have launched a new feature. ..."
</code>
```


or using `Artisan::call()`:



```
<code class="language-php">Artisan::call('tenants:run', [
    'commandname' =&gt; 'email:send', // String
    '--tenants' =&gt; ['8075a580-1cb8-11e9-8822-49c5d8f8ff23'] // Array
    '--option' =&gt; ['queue=1', 'subject=New Feature'] // Array
    '--argument' =&gt; ['body=We have launched a new feature.'] // Array
])
</code>
```


<h2 id="list"><strong>List</strong></h2>

The `tenants:list` command lists all existing tenants.



```
<code>php artisan tenants:list
Listing all tenants.
[Tenant] id: dbe0b330-1a6e-11e9-b4c3-354da4b4f339 @ localhost
[Tenant] id: 49670df0-1a87-11e9-b7ba-cf5353777957 @ dev.localhost
</code>
```


<h2 id="selectively-clearing-tenant-cache"><strong>Selectively clearing tenant cache</strong></h2>

You can delete specific tenants' cache by using the `--tags` option on `cache:clear`:



```
<code>php artisan cache:clear --tags=tenantdbe0b330-1a6e-11e9-b4c3-354da4b4f339
</code>
```


The tag is derived from `config('tenancy.cache.tag_base') . $id`.
