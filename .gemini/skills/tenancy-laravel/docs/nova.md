                                <h1 id="laravel-nova">Laravel Nova</h1>

<h2 id="in-the-central-app">In the central app</h2>

If you wish to use Laravel Nova in the central application (to manage tenants), you need to make a small change to the Nova migrations, they expect your model primary keys to always be unsigned big integers, but your tenants might be using `string` ids.



You can find the full Nova setup for managing tenants in the <a href="/saas-boilerplate">SaaS boilerplate</a>. It also implements Nova for <strong>both the central and tenant parts of the app, with separate resources</strong>.



<h2 id="in-the-tenant-app">In the tenant app</h2>

To use Nova inside of the tenant part of your application, do the following:



<ul>
<li>Publish the Nova migrations and move them to the `database/migrations/tenant` directory.



```
<code>php artisan vendor:publish --tag=nova-migrations
</code>
```
</li>
<li>Prevent Nova from adding its migrations to your central migrations by adding `Nova::ignoreMigrations()` to `NovaServiceProvider::boot()` (Don't do this if you want to use Nova <strong><a href="https://tenancyforlaravel.com/docs/v3/features/universal-routes">both in the central &amp; tenant parts</a></strong> of the app.)

</li>
<li>Add the tenancy middleware to your `nova.middleware` config. Example:



```
<code class="language-php">'middleware' =&gt; [
    // You can make this simpler by creating a tenancy route group
    InitializeTenancyByDomain::class,
    PreventAccessFromCentralDomains::class,
    'web',
    Authenticate::class,
    DispatchServingNovaEvent::class,
    BootTools::class,
    Authorize::class,
],
</code>
```
</li>
<li>In your `NovaServiceProvider`'s `routes()` method, replace the following lines:



```
<code class="language-php">-&gt;withAuthenticationRoutes()
-&gt;withPasswordResetRoutes()
</code>
```


with these lines:



```
<code class="language-php">-&gt;withAuthenticationRoutes([
    // You can make this simpler by creating a tenancy route group
    InitializeTenancyByDomain::class,
    PreventAccessFromCentralDomains::class,
    'nova',
])
-&gt;withPasswordResetRoutes([
    // You can make this simpler by creating a tenancy route group
    InitializeTenancyByDomain::class,
    PreventAccessFromCentralDomains::class,
    'nova',
])
</code>
```
</li>
<li>Set the `domain` in Nova config to `null`</li>
</ul>

<h2 id="file-thumbnails">Tenant file thumbnails and previews</h2>

To make the file field previews show correctly, you have to call `thumbnail(fn ($value, $disk) =&gt; tenant_asset($value))` and `preview(fn ($value, $disk) =&gt; tenant_asset($value))` on the field.



For example:



```
<code class="language-php">Avatar::make('Avatar', 'photo')
    -&gt;disk('public')
    -&gt;thumbnail(fn ($value, $disk) =&gt; tenant_asset($value)),
    -&gt;preview(fn ($value, $disk) =&gt; tenant_asset($value)),
</code>
```


Important: make sure to update `TenantAssetsController::$tenancyMiddleware` to the identification middleware of your choice.



E.g. if you're using InitializeByDomainOrSubdomain in your app, set `TenantAssetsController::$tenancyMiddleware` to InitializeByDomainOrSubdomain in TenancyServiceProvider's `boot()` method:



```
<code class="language-php">// In App\Providers\TenancyServiceProvider
public function boot(): void
{
    // ...
    TenantAssetsController::$tenancyMiddleware = InitializeTenancyByDomainOrSubdomain::class;
}
</code>
```
