                                <h1 class="x">Upgrading from 2.x</h1>

Note that most of the package's code is different now, which means that you will have to update all of the places where you interact with the package's classes.



That said, automatic tenancy will still work the same way.



<ul>
<li>Move all `_tenancy_` prefixed keys in tenant storage to `tenancy_` prefixed keys



Run this <strong>with 2.x code</strong> (but put application down first, this will break your app and require that you update to 3.x immediately after). Of course make a backup of the DB first.



(Note: You may want to make this part of a migration)



```
<code class="language-php">use Stancl\Tenancy\Tenant;
use Illuminate\Support\Str; 

tenancy()
    -&gt;all()
    -&gt;each(function (Tenant $tenant) {
        $keys = array_keys($tenant-&gt;data);
        $keys = array_filter($keys, function ($key) {
            return Str::startsWith($key, '_tenancy_');
        });

        foreach ($keys as $key) {
            $value = $tenant-&gt;data[$key];
            unset($tenant-&gt;data[$key]);
            $tenant-&gt;data[str_replace('_tenancy_', 'tenancy_', $key)] = $value;
        }

        $tenant-&gt;save();
    });

</code>
```
</li>
<li>Create migration file with `php artisan make:migration upgrade_tenancy_package`

</li>
<li>Replace your Http Kernel with a stock version (copy it from laravel/laravel: <a href="https://github.com/laravel/laravel/blob/master/app/Http/Kernel.php">https://github.com/laravel/laravel/blob/master/app/Http/Kernel.php</a>) and add back in any changes you made. The package now doesn't necessitate any Kernel changes, so remove all of the 2.x ones.

</li>
<li>Delete config, publish it &amp; the new files using `php artisan tenancy:install`. Register the new provider (see instructions on the <a href="https://tenancyforlaravel.com/docs/v3/installation">Installation</a> page)</li>
<li>Create Tenant model, as instructed on the <a href="https://tenancyforlaravel.com/docs/v3/tenants">Tenants</a> page</li>
<li>Update routes to use the correct middleware, see the <a href="https://tenancyforlaravel.com/docs/v3/routes">Routes</a> page</li>
</ul>
