                                <h1 id="tenant-config">Tenant config</h1>

It's likely you will need to use tenant-specific config in your application. That config could be API keys, things like "products per page" and many other things.



You could just use the tenant model to get these values, but you may still want to use Laravel's `config()` because of:



<ul>
<li>separation of concerns — if you just write tenancy implementation-agnostic `config('shop.products_per_page')`, you will have a much better time changing tenancy implementations</li>
<li>default values — you may want to use the tenant storage only to override values in your config file</li>
</ul>

<h2 id="enabling-the-feature"><strong>Enabling the feature</strong></h2>

Uncomment the following line in your `tenancy.features` config:



```
<code class="language-php">// Stancl\Tenancy\Features\TenantConfig::class,
</code>
```


<h2 id="configuring-the-mapping"><strong>Configuring the mapping</strong></h2>

This feature maps keys in the tenant storage (properties on the `Tenant` model, these may be <a href="https://tenancyforlaravel.com/docs/v3/tenants#custom-columns">custom columns</a> or virtual columns within `data`) to config keys based on the `$storageToConfigMap` <a href="https://tenancyforlaravel.com/docs/v3/configuration#static-properties">static property</a>.



For example, if your `$storageToConfigMap` looked like this:



```
<code class="language-php">\Stancl\Tenancy\Features\TenantConfig::$storageToConfigMap = [
    'paypal_api_key' =&gt; 'services.paypal.api_key',
];
</code>
```


the value of `paypal_api_key` in tenant model would be copied to the `services.paypal.api_key` config when tenancy is initialized.



<h2 id="mapping-the-value-to-multiple-config-keys">Mapping the value to multiple config keys</h2>

Sometimes you may want to copy the value to multiple config keys. To do that, specify the config keys as an array:



```
<code class="language-php">\Stancl\Tenancy\Features\TenantConfig::$storageToConfigMap = [
    'locale' =&gt; [
        'app.locale',
        'locales.default',
    ],
];
</code>
```
